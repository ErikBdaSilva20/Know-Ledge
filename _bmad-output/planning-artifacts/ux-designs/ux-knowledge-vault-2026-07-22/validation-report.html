A tight, well-scoped retrofit spine — every reported bug is real, every code citation spot-checked resolved exactly as claimed, and the "no new visual identity" discipline holds throughout. The pair's original weakness was that DESIGN.md's inheritance-by-reference stopped one step short (missing dark-mode tokens, four uncovered components, an unstated opacity value) — all now closed.

Accessibility lens materially shifted the picture: the diagnosis and fix direction were sound, but the 44px touch-target commitment wasn't operationalized against the actual controls, one row-action trigger has a confirmed, not hypothetical missing-`aria-label` bug in the live code, and the `(hover: hover)` gate had a real gap for hybrid touch+trackpad hardware. All three are now fixed in the spine: explicit sizing per control, a definitive (not hedged) aria-label commitment, and `(any-pointer: coarse)` in place of `(pointer: coarse)`.

Accessibility review (ad-hoc lens)

review-accessibility.md

The diagnosis and fix direction were sound — mutual-exclusive panes, viewport-capped textarea, always-reachable row actions, and a tap fallback for move all correctly eliminate the two reported bugs. But the accessibility floor was declared once in DESIGN.md and under-operationalized where an implementer would actually read instructions.

HighResolved

FolderMenu trigger has zero accessible name — confirmed, not hypothetical

Explorer.tsx:654
A bare <MoreHorizontal> icon inside a button with no aria-label/title — a live WCAG 4.1.2 failure today, independent of this audit. The spec had hedged ("verify at implementation time") despite already having this exact line open.
Fix: EXPERIENCE.md now states definitively: aria-label="Mais ações" (or per-item "Ações de {'{itemName}'}").
HighResolved

44px touch-target floor declared but not operationalized

DESIGN.md ↔ EXPERIENCE.md Component Patterns
Pane toggle, save button, and row-action trigger all stayed at their current ~32px shadcn sizes in the Fix columns — the token existed but wasn't wired to any of the three controls it named.
Fix: Each of the three rows now states min-h-11 min-w-11 below md: explicitly.
HighResolved

(hover: hover) gate has a hybrid-device gap

State Patterns
Devices with both a touchscreen and a trackpad (Surface/Chromebook-class) commonly report hover: hover, silently resurrecting the exact hover-only bug this audit exists to fix.
Fix: Switched to (any-pointer: coarse); residual risk documented explicitly in Accessibility Floor as a best-effort mitigation, not a guarantee.
MediumResolved

No announcement when the visible pane swaps

Editor.tsx pane toggle
A screen-reader user focused inside pane content (not the toggle button) gets a silent content swap with no notification.
Fix: Added an aria-live="polite" / role="status" state to State Patterns.
MediumResolved

Focus can drop to <body> when a pane unmounts

Editor.tsx conditional mount
If focus was inside the pane being hidden, React unmounting that subtree leaves the browser nothing to move focus to.
Fix: State Patterns now specifies focus moves to the newly-shown pane's toggle button on swap.
