- (decision) Fast path: no new visual identity — DESIGN.md inherits the existing shadcn/Tailwind v4 token set from knowledge/src/styles.css verbatim (oklch tokens, radius 0.625rem). Scope is behavioral/responsive only.
- (decision) Root cause of 'can't read documents on mobile': Editor.tsx defaults showRaw=true AND showPreview=true regardless of viewport; the grid only splits into 2 columns at md: (grid-cols-1 below), so both panes stack vertically, and the raw textarea auto-grows to its full content height (ref callback sets height = max(scrollHeight, parentHeight)) — burying the rendered preview far below the fold on phones.
- (decision) Same Editor.tsx is reused read-only by SharedDoc (shared-doc.tsx) — so the bug also blocks reading Base Compartilhada content on mobile for every role, not just personal docs.
- (decision) PaneToggle (Editor.tsx) already ships icon-only affordance below sm: (label text is 'hidden sm:inline') — this is nearly the user's requested mobile header already; the fix is state/default logic, not new UI.
- (assumption) [ASSUMPTION] Below md:, Editor defaults to preview-only (rendered view), since the user's stated goal is reading, and raw markdown-first contradicts that. User can flip to raw via the icon toggle; the two modes become mutually exclusive below md: (selecting one deselects the other) instead of both being independently toggleable.
- (decision) Secondary finding: Explorer.tsx (line 654) and FolderTree.tsx (line 143) reveal per-row actions (rename/delete/menu) via opacity-0 group-hover:opacity-100 — invisible and unreachable on touch (no hover state on mobile/tablet). FolderTree has a focus-within fallback, Explorer's row does not.
- (decision) Secondary finding: Explorer's move-doc-between-folders (drag documents between folders) uses native HTML5 draggable/onDragStart/onDrop — no touch/pointer polyfill, so it is desktop-only today.
- (assumption) [ASSUMPTION] AppShell.tsx (sidebar/header) and prose.css (rendered markdown: tables, images, code blocks) are already responsive-sound — verified via code read, no changes proposed there. Audit scope narrows to Editor/Workspace/Explorer plus a lighter pass on Admin list-row crowding.
